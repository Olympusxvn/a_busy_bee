# Contributing to A Busy Bee

Thanks for your interest in contributing! Every contribution helps the community save tokens.

## How to Contribute

### Report a Bug

1. Open an [issue](https://github.com/Olympusxvn/a_busy_bee/issues/new?template=bug_report.md)
2. Include reproduction steps
3. Describe expected vs actual behavior

### Request a Feature

1. Open an [issue](https://github.com/Olympusxvn/a_busy_bee/issues/new?template=feature_request.md)
2. Describe the problem it solves
3. Suggest an approach if you have one

### Submit a Pull Request

1. Fork the repository
2. Create a branch: `git checkout -b my-feature`
3. Make your changes
4. Test with Claude Code: `claude .`
5. Commit: `git commit -m "Add my feature"`
6. Push: `git push origin my-feature`
7. Open a Pull Request

### What Can You Contribute?

- New subagents (`.claude/subagents/`)
- New skills (`.claude/skills/`)
- Templates (`templates/`)
- Documentation improvements
- Bug fixes
- Token optimization strategies

## Guidelines

- Keep it simple - A Busy Bee values thriftiness
- Test your changes with Claude Code before submitting
- One PR per feature/fix
- Write clear commit messages

## First Time?

Look for issues labeled `good first issue` - these are designed for newcomers.

## Questions?

Open a [Discussion](https://github.com/Olympusxvn/a_busy_bee/discussions) or file an issue.
